# Plan: Remove Hardcoded Font Sizes (IDA Only)

## Problem

Qt widgets default to inheriting the application font, which is IDA Pro's configured font. However, several places in the code **hardcode explicit `font-size` values** that override this inheritance, preventing IDA's font from being used.

## Files with Hardcoded Font Sizes

### 1. `rikugan/ui/markdown.py`

| Line | Style | Current Value | Change To |
|------|-------|--------------|-----------|
| 82 | `_INLINE_CODE_STYLE` | `font-size:12px` | `font-size:inherit` |
| 88 | `_BLOCK_CODE_STYLE` | `font-size:12px` | `font-size:inherit` |
| 94 | `_BASE_INLINE_CODE_STYLE` | `font-size:12px` | `font-size:inherit` |
| 99 | `_BASE_BLOCK_CODE_STYLE` | `font-size:12px` | `font-size:inherit` |
| 110 | `_get_inline_code_style()` return | `font-size:12px` | `font-size:inherit` |
| 122 | `_get_block_code_style()` return | `font-size:12px` | `font-size:inherit` |
| 153 | Language tag | `font-size:10px` | `font-size:inherit` |
| 155 | Language tag (no theme) | `font-size:10px` | `font-size:inherit` |
| 194 | Heading sizes | `sizes = {1:18, 2:16, 3:14, 4:13}` | Remove hardcoded sizes, use `font-size:inherit` |

**Note on headings**: The heading sizes define visual hierarchy (h1=18px, h2=16px, etc.). If we change to `inherit`, all headings will be the same size. We should keep `font-weight:bold` for hierarchy but remove the hardcoded `font-size:{size}px`.

### 2. `rikugan/ui/panel_core.py`

| Line | Style | Current Value | Change To |
|------|-------|--------------|-----------|
| 51 | `_SMALL_BTN_STYLE` | `font-size: 11px` | `font-size: inherit` |
| 56 | `_CANCEL_BTN_STYLE` | `font-size: 11px` | `font-size: inherit` |

**Note on buttons**: These are small toolbar buttons (send, new, export, settings, etc.). Using `inherit` means they'll use the parent widget's font size, which is IDA's configured font.

## Implementation

### Step 1: Fix `rikugan/ui/markdown.py`

**Lines 80-101** — Change all `font-size:12px` to `font-size:inherit`:
- `_INLINE_CODE_STYLE` (line 82)
- `_BLOCK_CODE_STYLE` (line 88)
- `_BASE_INLINE_CODE_STYLE` (line 94)
- `_BASE_BLOCK_CODE_STYLE` (line 99)

**Lines 104-125** — Update `_get_inline_code_style()` and `_get_block_code_style()`:
- Change all `font-size:12px` to `font-size:inherit` in the returned strings

**Lines 153, 155** — Language tags:
- Change `font-size:10px` to `font-size:inherit`

**Line 194** — Headings:
- Change `font-size:{size}px` to `font-size:inherit`
- Remove the `sizes` dictionary or keep it unused

### Step 2: Fix `rikugan/ui/panel_core.py`

**Lines 49-58** — Button styles:
- `_SMALL_BTN_STYLE`: `font-size: 11px` → `font-size: inherit`
- `_CANCEL_BTN_STYLE`: `font-size: 11px` → `font-size: inherit`

# Plan: Add Manual Font Configuration Option

## Problem

After removing hardcoded fonts, the panel now inherits IDA Pro's application font (`Segoe UI`, `pointSize=9`). Users may want to override this with a custom font family and/or size for better readability.

## Goal

Add font family and font size controls to the Rikugan Settings dialog.

## Implementation

### Step 1: Add config fields

**File: `rikugan/core/config.py`**

Add two new fields to `RikuganConfig`:
- `font_family: str = ""` — empty means inherit from IDA (default)
- `font_size_override: int = 0` — 0 means inherit from IDA (default)

Add to `load()` serialization.
Add to `validate()` bounds checking:
- `font_family`: if non-empty, must be a valid string
- `font_size_override`: if non-zero, must be in range [6, 72]

### Step 2: Add UI controls to settings dialog

**File: `rikugan/ui/settings_dialog.py`**

In the Appearance section (or a new "Font" section):

1. **Font Family** — `QComboBox` with common monospace fonts as options:
   - `(Inherit from IDA)` (default, empty string)
   - `Consolas`
   - `Courier New`
   - `Lucida Console`
   - `Segoe UI`
   - `Monaco`
   - `Source Code Pro`
   
2. **Font Size** — `QSpinBox` with range [6, 72]:
   - Label: "Size:"
   - Default: 0 (Inherit)
   - When non-zero, overrides IDA's font size

**Note**: The font spinbox that was previously removed (lines 416-421) should be restored but with proper application logic.

### Step 3: Apply font to widgets

**File: `rikugan/ida/ui/panel.py`**

In `OnCreate()`, after `_apply_ida_theme()`, apply the configured font:

```python
# Apply custom font if configured
config = getattr(self._core, "_config", None)
if config and config.font_family:
    font = QApplication.instance().font()
    if config.font_family:
        font.setFamily(config.font_family)
    if config.font_size_override > 0:
        font.setPointSize(config.font_size_override)
    QApplication.instance().setFont(font)
```

Or apply per-widget instead of globally:
```python
if config.font_family or config.font_size_override > 0:
    font = self._core.font()
    if config.font_family:
        font.setFamily(config.font_family)
    if config.font_size_override > 0:
        font.setPointSize(config.font_size_override)
    self._core.setFont(font)
```

**Alternative**: Set font on specific widgets that need it (chat area, input, etc.) rather than globally to avoid affecting IDA's other widgets.

### Step 4: Wire up save/load

**File: `rikugan/ui/settings_dialog.py`**

In `_build_appearance_group()` (or new method):
- Create the font family combo and font size spinbox
- Connect to config

In `_on_accept()`:
- Save `font_family` and `font_size_override` to config

In `_load_current_values()`:
- Restore values from config
