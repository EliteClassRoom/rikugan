# Plan: Rikugan IDA Theme Integration Fix

## Executive Summary

Fix Rikugan to respect IDA Pro's theme (dark/light mode) instead of always using a hardcoded light theme.

---

## 1. Problem Analysis

- `panel_core.py:353` unconditionally sets `self.setStyleSheet(LIGHT_THEME)`
- IDA's `panel.py` tries to override with `generate_ida_theme_stylesheet()` afterward
- But Qt stylesheet cascade means the hardcoded default still takes precedence

### Current Code Flow

```
RikuganPanelCore.__init__()
  └─> _build_ui()
       └─> self.setStyleSheet(LIGHT_THEME)  // Line 353 - ALWAYS LIGHT!

IDA Panel wrapper:
  └─> _apply_ida_theme()
       └─> self._core.setStyleSheet(generate_ida_theme_stylesheet())  // Too late!
```

---

## 2. Solution

Add `set_theme(theme_name)` method to `RikuganPanelCore` and make each host apply its own theme.

### Theme Options

| Theme Name | Source | Description |
|------------|--------|-------------|
| `"ida"` | `ida/ui/panel.py` | Uses `generate_ida_theme_stylesheet()` to match IDA's colors |
| `"binja"` | `binja/ui/panel.py` | Uses Binary Ninja's native theme |
| `"dark"` | `ui/styles.py:DARK_THEME` | VS Code Dark+ inspired |
| `"light"` | `ui/styles.py:LIGHT_THEME` | Monokai Pro Light inspired |

---

## 3. Implementation Steps

### Step 1: Modify `ui/panel_core.py`

**File**: `rikugan/ui/panel_core.py`

**Change**: 
1. Remove `self.setStyleSheet(LIGHT_THEME)` from `_build_ui()`
2. Add `set_theme(theme_name: str)` method

```python
def set_theme(self, theme: str) -> None:
    """Set the UI theme.
    
    Args:
        theme: One of 'ida', 'binja', 'dark', or 'light'.
    """
    if theme == "ida":
        # IDA theme requires the host to provide the stylesheet
        # This will be called by ida/ui/panel.py after initialization
        return
    elif theme == "binja":
        # Binary Ninja theme - similar handling
        return
    elif theme == "dark":
        self.setStyleSheet(DARK_THEME)
    else:
        self.setStyleSheet(LIGHT_THEME)
```

### Step 2: Modify `ida/ui/panel.py`

**File**: `rikugan/ida/ui/panel.py`

**Change**: In `_apply_ida_theme()`, replace the fallback logic with direct theme application.

Current problematic code (lines 546-560):
```python
def _apply_ida_theme(self) -> None:
    try:
        theme_stylesheet = generate_ida_theme_stylesheet()
        if self._core:
            self._core.setStyleSheet(theme_stylesheet)  # Overrides but flow is wrong
    except Exception as e:
        from rikugan.ui.styles import DARK_THEME
        if self._core:
            self._core.setStyleSheet(DARK_THEME)  # Fallback is DARK, not IDA-aware
```

Should become:
```python
def _apply_ida_theme(self) -> None:
    """Apply IDA Pro's theme to the Rikugan panel."""
    try:
        theme_stylesheet = generate_ida_theme_stylesheet()
        if self._core:
            self._core.setStyleSheet(theme_stylesheet)
    except Exception as e:
        log_error(f"Failed to apply IDA theme: {e}")
        # Don't silently fall back to DARK_THEME - keep trying IDA theme
        # or use a neutral base that will be corrected on next attempt
```

### Step 3: Modify `binja/ui/panel.py`

**File**: `rikugan/binja/ui/panel.py`

**Change**: Apply Binary Ninja's theme similarly.
